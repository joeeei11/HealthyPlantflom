## 一、部署与启动

### 1.1 环境要求

| 依赖 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | >= 18 | 后端运行时（含 `fetch` 原生支持） |
| npm | >= 9 | 包管理器 |
| MySQL | 8.0 | 数据库 |
| Docker（可选） | >= 20 | 用于一键启动 MySQL |

### 1.2 配置环境变量

#### backend/.env

复制 `backend/.env.example` 并重命名为 `backend/.env`，填写以下字段：

```env
DB_HOST=localhost          # 数据库主机，Docker 方式填 localhost
DB_PORT=3306               # 数据库端口，默认 3306
DB_NAME=mental_health      # 数据库名称，保持不变
DB_USER=root               # 数据库用户名
DB_PASSWORD=               # 数据库密码（自行设置）
JWT_SECRET=                # JWT 签名密钥，建议使用随机长字符串（32位以上）
JWT_EXPIRES_IN=7d          # Token 有效期，默认 7 天
DEEPSEEK_API_KEY=          # DeepSeek API 密钥，用于 AI 心理助手功能
DEEPSEEK_BASE_URL=https://api.deepseek.com  # DeepSeek 接口地址，保持不变
PORT=3000                  # 后端服务端口
```

> **注意：** `DEEPSEEK_API_KEY` 留空时，AI 心理助手功能将返回"AI 服务未配置"错误。
> 其他功能（预约、咨询等）不受影响。

#### frontend/.env

复制 `frontend/.env.example` 并重命名为 `frontend/.env`，填写以下字段：

```env
VITE_API_BASE_URL=http://localhost:3000   # 后端接口地址
```

### 1.3 启动步骤

#### 方案 A：使用 Docker 启动 MySQL（推荐）

```bash
# 1. 在项目根目录启动 MySQL 容器（会自动执行 database/migrations/ 下的 SQL 初始化数据库）
docker-compose up -d

# 2. 验证 MySQL 容器启动成功
docker ps   # 应看到 mental_health_mysql 容器状态为 Up
```

默认使用环境变量 `MYSQL_ROOT_PASSWORD=changeme`、`MYSQL_USER=app`、`MYSQL_PASSWORD=changeme`。
生产环境务必修改 `docker-compose.yml` 中的密码或在启动前设置对应环境变量。

#### 方案 B：手动初始化数据库

若已有本地 MySQL 服务：

```bash
# 创建数据库（如尚未创建）
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS mental_health CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 执行初始化 migration
mysql -u root -p mental_health < database/migrations/001_init.sql
```

#### 启动后端

```bash
cd backend
npm install       # 首次运行时安装依赖
npm run dev       # 开发模式（nodemon 热重载），监听 3000 端口
# 或
npm start         # 生产模式
```

#### 启动前端

```bash
cd frontend
npm install       # 首次运行时安装依赖
npm run dev       # 开发模式，监听 5173 端口
```

### 1.4 验证启动成功

**验证后端：**

```bash
curl http://localhost:3000/health
# 期望返回：{"code":200,"data":null,"message":"ok"}
```

**验证前端：**

浏览器访问 `http://localhost:5173`，应自动跳转到 `/login` 登录页面。

## 二、学生端使用说明

学生端访问地址：`http://localhost:5173/student`（未登录时自动跳转到登录页）

### 2.1 注册

1. 打开 `http://localhost:5173`，进入登录页面
2. 点击"注册"，填写以下信息：
   - **用户名**：唯一标识，登录后显示
   - **邮箱**：用于登录（唯一）
   - **密码**：至少 6 位
   - **角色**：选择"学生"（`student`）
3. 注册成功后自动登录并跳转到学生端首页

### 2.2 登录

1. 在登录页面输入注册时使用的**邮箱**和**密码**
2. 登录成功后，系统根据账号角色自动跳转到对应端首页
   - 学生 → `/student`
   - 咨询师 → `/counselor`
   - 管理员 → `/school`

### 2.3 完善个人档案

首次使用建议先完善个人档案（`/student/profile`）：

- **真实姓名**（必填，首次保存时必须提供）
- 学号、性别、年级、专业、学院
- 手机号、紧急联系人及电话

> 个人档案信息仅供咨询预约使用，不会对外公开。

### 2.4 查看咨询师列表与详情

1. 进入"咨询师列表"页（`/student/counselors`）
2. 页面仅展示**当前接受预约**且**账号已启用**的咨询师
3. 支持按咨询师姓名关键字搜索
4. 每位咨询师展示：姓名、职称、擅长方向、个人简介、可预约时段、每日最大接诊数

### 2.5 发起预约

1. 在咨询师列表找到目标咨询师后，点击"预约"
2. 填写预约信息：
   - **预约日期**：YYYY-MM-DD 格式
   - **开始时间 / 结束时间**：HH:MM 格式，开始时间须早于结束时间
   - **咨询方式**：`online`（线上）或 `offline`（线下，默认）
   - **预约原因**：选填
3. 提交后预约状态为 `pending`（待确认），等待咨询师确认
4. **注意事项：**
   - 若该时段已被其他预约占用，提交会失败（`409 该时段已有其他预约`）
   - 若当日预约数已达咨询师上限，提交会失败

### 2.6 查看我的预约

进入"我的预约"页（`/student/appointments`），可查看所有预约记录，支持按状态筛选：

| 状态 | 含义 |
|------|------|
| `pending` | 待咨询师确认 |
| `confirmed` | 已确认，等待赴约 |
| `cancelled` | 已取消 |
| `completed` | 已完成 |
| `no_show` | 爽约 |

### 2.7 取消预约

在"我的预约"页，对状态为 `pending` 或 `confirmed` 的预约可点击"取消"：
- 可填写取消原因（选填）
- 已取消的预约**不可恢复**

### 2.8 提交咨询评价

咨询会话完成后（状态为 `completed`），可对该次咨询进行评价：
- **评分**：1-5 星（必填）
- **评价内容**：文字说明（选填）
- **是否匿名**：勾选后咨询师看不到评价人姓名
- 每次会话只能评价一次，不可修改

### 2.9 使用 AI 心理助手

进入"AI 心理助手"页（`/student/ai`）：

1. 点击"新建对话"创建一个对话（可自定义标题，留空则自动以首条消息截取）
2. 在输入框输入你的问题或心事，发送后 AI 以**流式输出**方式逐字显示回复
3. AI 助手名为"心语"，提供情感支持和心理疏导
4. 历史对话保存在对话列表中，可随时继续
5. **重要：** AI 助手不能替代专业心理咨询。若有自伤或自杀念头，请立即联系专业咨询师或拨打全国心理援助热线：**400-161-9995**

## 三、咨询师端使用说明

咨询师账号由学校管理员创建，无法自行注册。

咨询师端访问地址：`http://localhost:5173/counselor`

### 3.1 登录

使用管理员分配的邮箱和初始密码登录。登录成功后跳转到咨询师首页（`/counselor`）。

### 3.2 完善个人档案

首次登录后，请先完善个人档案（`/counselor/profile`），信息会展示在学生端的咨询师列表中：

| 字段 | 说明 |
|------|------|
| 真实姓名 | **首次保存必填** |
| 性别 | `male` / `female` / `other` |
| 职称 | 如"副教授"、"心理咨询师" |
| 资质证书 | 资质描述文字 |
| 擅长方向 | JSON 数组，如 `["焦虑","抑郁","人际关系"]` |
| 个人简介 | 向学生展示的介绍 |
| 可预约时段 | JSON 结构，如 `{"Mon":["09:00-10:00","14:00-15:00"],"Tue":["10:00-11:00"]}` |
| 每日最大接诊数 | 默认 8，超出后学生无法预约当日时段 |
| 工作电话 | 选填 |
| 是否接受预约 | 设为 `false` 后不再在学生端显示 |

### 3.3 处理预约

进入"预约管理"页（`/counselor/appointments`），查看所有学生发来的预约请求：

**确认预约：**
- 对状态为 `pending` 的预约，点击"确认"即可
- 确认后预约状态变为 `confirmed`

**拒绝预约：**
- 对状态为 `pending` 的预约，点击"拒绝"
- 可填写拒绝原因，预约状态变为 `cancelled`

> 只能操作状态为 `pending` 的预约，已确认或已取消的预约无法再次操作。

### 3.4 开始咨询会话

预约到时后，在"会话管理"页（`/counselor/sessions`）点击"开始会话"：

- **基于预约开始**：传入 `appointmentId`（须为 `confirmed` 状态的预约），同时将预约标记为 `completed`
- **临时安排**：不传 `appointmentId`，直接传 `studentId` 开始临时会话

> 同一预约只能关联一个会话，重复操作会报错。

### 3.5 结束会话

在会话详情页（`/counselor/sessions/:id`）点击"结束会话"，会话状态变为 `completed`，同时记录结束时间。

### 3.6 编写案例笔记

会话进行中或结束后，可添加案例笔记（对学生不可见，仅咨询师本人可查）：

| 字段 | 说明 |
|------|------|
| 内容 | 笔记正文（必填） |
| 风险等级 | `low`（低）/ `medium`（中）/ `high`（高）/ `crisis`（危机），默认 `low` |
| 是否需要跟进 | 布尔值，默认否 |
| 是否仅私有 | 默认为私有（`true`），仅本咨询师可见 |

### 3.7 查看会话历史

"会话列表"页展示所有历史会话，支持按状态筛选（`in_progress` / `completed`），点击具体会话可查看详情及笔记。

## 四、学校管理端使用说明

管理端账号需在数据库中手动插入（`role = 'school'`），或由已有管理账号创建。

管理端访问地址：`http://localhost:5173/school`

### 4.1 登录

使用管理员邮箱和密码登录。

### 4.2 查看数据统计

首页（`/school`）展示平台总览数据：

| 指标 | 说明 |
|------|------|
| 学生总数 | 账号启用的学生用户数 |
| 咨询师总数 | 账号启用的咨询师用户数 |
| 总预约数 | 所有预约记录数 |
| 已完成会话数 | 状态为 `completed` 的会话数 |
| 平均评分 | 所有评价的平均星级（精确到小数点后2位） |

### 4.3 添加咨询师账号

进入"咨询师管理"页（`/school/counselors`），点击"添加咨询师"：

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 用户名 | 必填 | 登录用户名，需唯一 |
| 邮箱 | 必填 | 登录邮箱，需唯一 |
| 密码 | 必填 | 初始密码，建议告知咨询师后自行修改 |
| 真实姓名 | 选填 | 提供后自动创建咨询师档案 |

### 4.4 启用/禁用咨询师

在咨询师列表中，对每位咨询师可切换账号状态：
- **禁用**：咨询师无法登录，学生端也不再显示该咨询师
- **启用**：恢复正常

### 4.5 发布公告

进入"公告管理"页（`/school/announcements`）：

**创建公告：**

| 字段 | 说明 |
|------|------|
| 标题 | 必填 |
| 内容 | 必填 |
| 目标受众 | `all`（全部）/ `student`（仅学生）/ `counselor`（仅咨询师），默认 `all` |
| 是否置顶 | 置顶公告排在最前 |
| 是否立即发布 | `true` 立即发布；`false` 保存为草稿 |
| 过期时间 | 选填，UTC 时间格式，留空表示永不过期 |

**删除公告：** 在公告列表点击"删除"，操作不可撤销。

> 当前版本暂未实现公告的编辑功能（更新已发布公告）。

## 五、API 接口说明

### 通用说明

**Base URL：** `http://localhost:3000`

**统一响应格式：**
```json
// 成功
{ "code": 200, "data": { ... }, "message": "success" }

// 失败
{ "code": 400, "data": null, "message": "错误描述" }
```

**认证方式：** 在请求头中携带 JWT Token：
```
Authorization: Bearer <your_token>
```

**健康检查：**
```
GET /health
```

### 5.1 认证模块（auth）

所有认证接口无需携带 Token。

#### 注册

```
POST /api/v1/auth/register
```

请求体：
```json
{
  "username": "zhangsan",
  "email": "zhangsan@example.com",
  "password": "password123",
  "role": "student"
}
```

> `role` 取值：`student` / `counselor` / `school`

响应示例：
```json
{
  "code": 201,
  "data": {
    "token": "eyJhbGci...",
    "user": {
      "id": 1,
      "username": "zhangsan",
      "email": "zhangsan@example.com",
      "role": "student",
      "isActive": true,
      "avatarUrl": null,
      "lastLoginAt": null,
      "createdAt": "2026-03-09T00:00:00.000Z"
    }
  },
  "message": "注册成功"
}
```

#### 登录

```
POST /api/v1/auth/login
```

请求体：
```json
{
  "email": "zhangsan@example.com",
  "password": "password123"
}
```

响应示例：
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGci...",
    "user": { "id": 1, "username": "zhangsan", "role": "student", ... }
  },
  "message": "登录成功"
}
```

#### 刷新 Token

```
POST /api/v1/auth/refresh
```

请求体：
```json
{
  "token": "eyJhbGci..."
}
```

响应示例：
```json
{
  "code": 200,
  "data": { "token": "eyJhbGci...（新 token）" },
  "message": "Token 刷新成功"
}
```

#### 获取当前用户信息

```
GET /api/v1/auth/me
```

需要认证：是

响应示例：
```json
{
  "code": 200,
  "data": { "id": 1, "username": "zhangsan", "role": "student", ... },
  "message": "success"
}
```

#### 发起预约

```
POST /api/v1/student/appointments
```

请求体：
```json
{
  "counselorId": 2,
  "appointmentDate": "2026-03-15",
  "startTime": "09:00",
  "endTime": "10:00",
  "type": "offline",
  "reason": "学业压力较大，希望获得心理疏导"
}
```

> `type` 取值：`online` / `offline`（默认 `offline`）

错误情况：
- `400` 缺少必要字段
- `400` 开始时间晚于结束时间
- `400` 咨询师当前不接受预约或当日已满
- `404` 咨询师不存在
- `409` 时段冲突

#### 获取我的预约列表

```
GET /api/v1/student/appointments?status=&page=1&pageSize=10
```

查询参数：
- `status`（选填）：`pending` / `confirmed` / `cancelled` / `completed` / `no_show`
- `page`（选填，默认 1）
- `pageSize`（选填，默认 10）

响应示例：
```json
{
  "code": 200,
  "data": {
    "total": 5,
    "page": 1,
    "pageSize": 10,
    "list": [
      {

        "studentId": 1,
        "counselorId": 2,
        "appointmentDate": "2026-03-15",
        "startTime": "09:00:00",
        "endTime": "10:00:00",
        "type": "offline",
        "status": "pending",
        "reason": "学业压力较大",
        "cancelReason": null,
        "counselor": {
          "id": 2,
          "username": "lisi",
          "avatarUrl": null,
          "counselorProfile": { "realName": "李四", "title": "心理咨询师" }
        }
      }
    ]
  },
  "message": "success"
}
```

#### 取消预约

```
DELETE /api/v1/student/appointments/:id
```

请求体（选填）：
```json
{
  "cancelReason": "时间冲突，无法赴约"
}
```

只能取消状态为 `pending` 或 `confirmed` 的预约。

#### 提交咨询评价

```
POST /api/v1/student/feedback
```

请求体：
```json

```


响应示例：
```json
{
  "code": 200,
  "data": [
    { "id": 1, "conversationId": 1, "role": "user", "content": "我最近压力很大", "createdAt": "..." },
    { "id": 2, "conversationId": 1, "role": "assistant", "content": "我听到你说压力很大...", "tokensUsed": 150, "createdAt": "..." }
  ],

}
```

#### 发送消息（SSE 流式输出）

```
POST /api/v1/student/ai/conversations/:id/messages
Content-Type: application/json
```

请求体：
```json
{
  "content": "我最近考试压力很大，睡眠也不好"
}
```

**响应类型：** `text/event-stream`（SSE）

SSE 事件格式：
```
# AI 逐字回复（多个 delta 事件）
data: {"type":"delta","content":"我"}
data: {"type":"delta","content":"听"}
data: {"type":"delta","content":"到"}

# 回复完成
data: {"type":"done"}

# 错误（AI 未配置、网络失败等）
data: {"type":"error","message":"AI 服务未配置，请联系管理员"}
```

前端使用 `EventSource` 或手动读取 `ReadableStream` 接收流式数据。

## 六、常见问题排查

### 6.1 数据库连接失败

**错误现象：** 后端启动时输出 `[server] startup failed` 或 `SequelizeConnectionError`。

**排查步骤：**

1. 确认 MySQL 服务正在运行：
   ```bash
   # Docker 方式
   docker ps  # 检查 mental_health_mysql 容器状态
   docker logs mental_health_mysql  # 查看容器日志

   # 本地 MySQL
   mysql -u root -p -e "SELECT 1"
   ```

2. 检查 `backend/.env` 中的数据库配置是否正确：
   - `DB_HOST`、`DB_PORT`、`DB_NAME`、`DB_USER`、`DB_PASSWORD`

3. 确认数据库 `mental_health` 已创建：
   ```bash
   mysql -u root -p -e "SHOW DATABASES;"
   ```

4. 确认 migration 已执行（表已创建）：
   ```bash
   mysql -u root -p mental_health -e "SHOW TABLES;"
   ```

### 6.2 JWT Token 过期 / 无效

**错误现象：** API 返回 `{ "code": 401, "message": "Token 已过期" }` 或 `"Token 无效"`。

**解决方法：**

- **Token 过期**：调用刷新接口重新获取 Token：
  ```
  POST /api/v1/auth/refresh
  Body: { "token": "<过期的 token>" }
  ```
  > 注意：只能用**有效**的 Token 换取新 Token，过期 Token 无法刷新，需重新登录。

- **Token 无效**：重新调用登录接口获取新 Token。

- **前端未携带 Token**：检查请求头是否包含 `Authorization: Bearer <token>`。

### 6.3 AI 接口无响应

**错误现象：** AI 心理助手发送消息后无任何输出，或 SSE 立即返回 error 事件。

**排查步骤：**

1. 检查 `backend/.env` 中 `DEEPSEEK_API_KEY` 是否已填写（不能是默认值 `your_deepseek_api_key_here`）

2. 检查 `DEEPSEEK_BASE_URL` 是否可访问（默认 `https://api.deepseek.com`）

3. 查看 SSE 错误事件内容：
   - `"AI 服务未配置，请联系管理员"` → API Key 未配置
   - `"AI 服务暂时不可用，请稍后重试"` → DeepSeek API 返回非 200 状态码（检查 Key 是否有效、余额是否充足）
   - `"AI 服务连接失败，请检查网络"` → 网络不通，检查服务器出口网络

4. 验证 API Key 有效性：
   ```bash
   curl https://api.deepseek.com/v1/models \
     -H "Authorization: Bearer <your_api_key>"
   ```

### 6.4 前端跨域报错

**错误现象：** 浏览器控制台出现 `CORS policy` 错误，API 请求被阻止。

**排查步骤：**

1. 检查后端 `app.js` 中的 CORS 配置：
   ```js
   app.use(cors({
     origin: process.env.FRONTEND_URL || 'http://localhost:5173',
     credentials: true,
   }));
   ```

2. 如果前端运行在非 `5173` 端口，在 `backend/.env` 中添加：
   ```env
   FRONTEND_URL=http://localhost:你的端口号
   ```

3. 确认前端 `frontend/.env` 中 `VITE_API_BASE_URL` 与后端地址一致：
   ```env
   VITE_API_BASE_URL=http://localhost:3000
   ```

4. 重启后端服务使配置生效。

### 6.5 预约冲突处理

**错误现象：** 发起预约时返回 `{ "code": 409, "message": "该时段已有其他预约，请选择其他时间" }`。

**说明：** 系统会检测预约时段重叠（区间 `[startTime, endTime)` 与已有预约重叠即视为冲突）。

**解决方法：**
- 选择其他时间段
- 参考咨询师的 `availableSlots` 字段中公示的可用时段
- 若当日预约已满（返回 `"该咨询师当日预约已满"`），改选其他日期

**其他预约相关错误：**

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `该咨询师当前不接受预约` | 咨询师已关闭接受预约 | 选择其他咨询师 |
| `当前预约状态为 xxx，无法取消` | 只有 `pending` 或 `confirmed` 状态可取消 | 无法取消 |
| `当前预约状态为 xxx，无法操作` | 咨询师只能操作 `pending` 状态的预约 | 无法操作 |
